# Evidence Construction Notes

## Purpose

This stage converts model predictions and local SHAP explanations into
structured evidence packets for selected TP, FN, FP, and TN cases.

The evidence packet is the bridge between:

```text
model + SHAP
→ structured patient-level evidence
→ LLM prompt
→ validated explanation
```

## Inputs

The refreshed evidence packets use the final model pipeline:

- final preprocessor: `models/icu_preprocessor.pkl`
- final model: `models/lgbm_tuned_clean.pkl`
- final threshold: `models/lgbm_tuned_clean_threshold.json`
- selected local cases: `reports/02_explainability/tables/selected_local_cases.csv`

The evidence packets are regenerated by:

```text
scripts/19_refresh_evidence_packets.py
```

## Packet Structure

Each evidence packet contains:

- patient label
- original test row index
- predicted label
- predicted mortality probability
- final threshold
- true label and prediction type for internal evaluation only
- top risk-increasing SHAP evidence
- top risk-decreasing SHAP evidence
- clinical meaning for known features
- caution flags for values requiring careful interpretation

The LLM prompt does **not** receive the true label or TP/FN/FP/TN case type.
Those fields stay in the evidence packet for audit and reporting only.

## Example

For the selected TP case, risk-increasing evidence includes:

- `d1_heartrate_min = 0`
- high lactate
- low minimum SpO2
- diagnosis category information
- low GCS motor score
- low arterial pH
- mechanical ventilation
- low systolic blood pressure

The zero-valued minimum heart rate receives a caution flag because it may
represent either an extreme clinical event or a recording artifact.

## Why This Matters

Raw SHAP tables are useful for model debugging, but they are not ideal as direct
LLM input. The evidence packet makes the explanation input explicit and
auditable:

- every feature has a value,
- every feature has a SHAP direction,
- known clinical meanings are attached,
- caution flags are preserved,
- unsupported clinical inference is discouraged.

This structure allows the later deterministic validator to check whether the LLM
explanation stayed grounded in the model evidence.
